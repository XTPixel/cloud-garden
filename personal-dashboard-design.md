# 个人仪表盘 / 数字花园设计文档

## 1. 项目定位

这个项目不是一个传统意义上的“个人主页”，而是一个每天都会打开、能反映个人状态与兴趣的私人入口。它同时具备三种属性：

- **个人仪表盘**：承载天气、时间、常用链接、灵感记录等高频信息。
- **数字花园**：记录灵感、偏好、阅读、音乐、电影、语录等长期生长的内容。
- **情绪空间**：通过天气、动效、宠物、植物、配色和文字，让页面呈现“我当下的气息”。

目标不是做一个功能很多的工具站，而是做一个轻量、温暖、可持续维护的个人空间。

## 2. 设计关键词

- **私密感**：像自己的桌面，而不是给别人看的作品集。
- **低打扰**：信息要少而精，不制造焦虑。
- **可生长**：内容可以慢慢增加，比如语录库、推荐列表、便签墙。
- **有情绪**：页面会根据时间、天气、心境发生轻微变化。
- **本地优先**：早期尽量使用 `localStorage` 保存数据，降低后端复杂度。

## 3. 用户场景

### 3.1 每天打开浏览器时

用户希望快速看到天气、时间、今日重点事项，并从常用书签进入工作或学习网站。

### 3.2 想快速记录灵感时

用户希望在页面上直接写一张小便签，保存一个想法、标题、句子或灵感片段。

### 3.3 需要一点随机灵感时

用户点击“今日份推荐”，获得一首歌、一本书、一部电影或一句话，让页面产生一点陪伴感。

### 3.4 想让网页更像自己时

用户可以维护自己的语录、常用快捷语、书签、推荐清单和倒数日，让页面逐渐变成个人化空间。

## 4. 信息架构

页面可以按三组模块组织：

1. **时空与心境：氛围组**
   - 动态天气卡片
   - 自定义箴言
   - 极简时钟 / 日历
   - 倒数日或年度进度

2. **高频实用工具：效率组**
   - 快速书签 / 导航格
   - 剪贴板快捷语

3. **趣味与彩蛋：灵魂组**
   - 像素宠物或动态植物
   - 今日份随机推荐
   - 留言便签墙

## 5. 页面布局建议

### 5.1 桌面端布局

建议使用三栏或 Bento Grid 风格布局：

```text
┌──────────────────────────────────────────────┐
│ 顶部：问候语 + 当前日期 + 主题切换             │
├───────────────┬────────────────┬─────────────┤
│ 天气卡片       │ 翻页时钟 / 日历  │ 箴言卡片     │
├───────────────┴────────────────┴─────────────┤
│ 快速导航格：GitHub / Gmail / Notion / B站 ... │
├─────────────────────┬────────────────────────┤
│ 剪贴板快捷语          │ 今日推荐 / 灵感入口       │
├─────────────────────┴────────────────────────┤
│ 便签墙 / 随机推荐 / 像素宠物                   │
└──────────────────────────────────────────────┘
```

### 5.2 移动端布局

移动端建议改为单列卡片流：

1. 问候语
2. 天气 + 时间
3. 快速导航
4. 剪贴板快捷语
5. 箴言 / 随机推荐
6. 便签墙
7. 像素宠物或动态植物

移动端不需要强行展示所有内容，可以允许部分模块折叠。

## 6. 模块设计

## 6.1 动态天气卡片

### 功能说明

显示用户所在城市的实时天气，包括：

- 城市名
- 温度
- 天气状态
- 体感温度
- 湿度 / 风力
- 天气图标或动态背景

### 交互设计

- 默认显示一个预设城市。
- 支持在配置文件中修改城市。
- 天气状态改变时，卡片背景同步变化：
  - 晴天：暖色渐变、柔和光斑
  - 阴天：灰蓝色背景、低对比度
  - 雨天：深蓝背景、雨滴动效
  - 雪天：浅色背景、雪花动效
  - 夜晚：深色背景、星点装饰

### 技术建议

- 天气 API：和风天气、OpenWeatherMap 或其他免费 API。
- 初期可以先使用 mock 数据，等页面稳定后再接 API。
- API Key 不建议直接硬编码在前端仓库中；如果只是个人本地使用，可以先放在 `.env` 中。

### 数据结构示例

```js
const weather = {
  city: '杭州',
  temperature: 26,
  condition: 'rainy',
  humidity: 78,
  wind: '东北风 2级'
}
```

## 6.2 一句自定义箴言

### 功能说明

展示一句当前喜欢的话，可以点击切换随机语录。

### 内容类型

- 自己写的话
- 喜欢的书摘
- 阶段性提醒
- 简短目标
- 心情记录

### 交互设计

- 点击卡片随机切换一句。
- 可选：增加“固定今日箴言”功能，每天只随机一次。
- 可选：增加喜欢按钮，把某句话加入高频展示列表。

### 数据结构示例

```js
const quotes = [
  {
    text: '慢慢来，比较快。',
    source: '个人语录'
  },
  {
    text: '把注意力放回手上的事情。',
    source: '每日提醒'
  }
]
```

## 6.3 极简时钟 / 日历

### 功能说明

用设计感强的方式展示当前时间、日期和个人进度。

### 可选形态

- 翻页时钟
- 环形进度日历
- 年度进度条
- 本月进度条
- 倒数日卡片

### 推荐内容

- 当前时间
- 今天是星期几
- 今年已经过去百分之几
- 距离某个重要日子还有几天
- 今年已读书数量

### 示例文案

```text
20:45
星期三 · 7 月 1 日
2026 年已经过去 49.6%
距离发薪日还有 9 天
今年已读 12 本书
```

## 6.4 快速书签 / 导航格

### 功能说明

替代浏览器默认书签栏，作为个人常用网站启动台。

### 默认分类

- 工作：GitHub、Gmail、Notion、Google Drive
- 学习：YouTube、B站、Coursera、文档站
- 创作：Figma、Canva、写作工具
- 娱乐：Spotify、豆瓣、Letterboxd

### 交互设计

- 每个网站使用图标 + 名称 + 简短描述。
- 支持 hover 动效，如轻微上浮、发光边框。
- 可选：支持按分类筛选。
- 可选：支持键盘快捷键打开常用网站。

### 数据结构示例

```js
const bookmarks = [
  {
    title: 'GitHub',
    url: 'https://github.com',
    icon: 'github',
    group: 'work'
  },
  {
    title: 'B站',
    url: 'https://www.bilibili.com',
    icon: 'bilibili',
    group: 'fun'
  }
]
```

## 6.5 剪贴板快捷语

### 功能说明

保存常用文本，点击即可复制到剪贴板。

### 适合内容

- 邮件签名
- 自我介绍
- 常用回复
- 地址信息
- 提示词模板
- 项目介绍

### 交互设计

- 每条快捷语显示标题，不直接展示全部内容。
- 点击后复制，并显示“已复制”。
- 可选：长按或点击编辑按钮查看完整内容。

### 技术建议

使用浏览器 Clipboard API：

```js
await navigator.clipboard.writeText(text)
```

## 6.6 像素宠物或动态植物

### 功能说明

页面角落放一个小型动态角色，让页面更有生命感。

### 方案 A：像素宠物

- 使用 CSS 或 Canvas 绘制。
- 默认待机动画：眨眼、晃动、呼吸。
- 鼠标靠近时看向鼠标。
- 点击时显示一句小台词。

### 方案 B：动态植物

- 根据访问天数、记录便签数量或每日打开次数成长。`n- 写下一张新便签后，植物长出一片叶子或开花。
- 长期访问页面会形成自己的“小花园”。

### 推荐实现顺序

第一版先做简单 CSS 动画宠物，后续再升级为 Canvas 或 SVG 动画。

## 6.7 今日份随机推荐

### 功能说明

点击按钮后随机推荐一个内容：电影、书、音乐或文章。

### 推荐类型

- 电影
- 书籍
- 音乐
- 播客
- 文章
- 游戏
- 个人小行动

### 交互设计

- 点击“给我一点灵感”。
- 卡片翻转或淡入展示推荐结果。
- 可选：每天固定一个推荐，避免无限刷新。
- 可选：增加“我看过了 / 不感兴趣”状态。

### 数据结构示例

```js
const recommendations = [
  {
    type: 'book',
    title: '禅与摩托车维修艺术',
    note: '适合在想重新理解生活节奏时读。'
  },
  {
    type: 'music',
    title: '坂本龙一 - Merry Christmas Mr. Lawrence',
    note: '适合夜晚写东西。'
  }
]
```

## 6.8 留言便签墙

### 功能说明

记录临时灵感、短句、想法和备忘。

### 交互设计

- 新建一张便签。
- 支持编辑、删除。
- 支持不同颜色。
- 支持拖拽排序作为后续增强。
- 数据先保存到 `localStorage`。

### 视觉风格

- 像贴在墙上的便利贴。
- 可以有轻微旋转角度。
- 不需要过度规整，保留一点手作感。

## 7. 视觉风格

## 7.1 整体风格方向

推荐采用“温暖极简 + Bento Grid + 微动效”的方向。

### 关键词

- 柔和渐变
- 半透明玻璃拟态
- 圆角卡片
- 轻微阴影
- 微妙噪点纹理
- 低饱和色彩

## 7.2 色彩建议

### 日间主题

- 背景：米白、浅灰、淡蓝
- 主色：湖蓝、鼠尾草绿、暖橙
- 文字：深灰而非纯黑

### 夜间主题

- 背景：深蓝黑、暗紫、炭灰
- 主色：霓虹蓝、柔紫、月光黄
- 文字：低亮度白色

### 天气联动主题

```text
sunny  -> warm / orange / gold
rainy  -> blue / slate / mist
cloudy -> gray / lavender / muted
snowy  -> white / cyan / soft
night  -> navy / purple / star
```

## 7.3 字体建议

- 中文：系统默认、霞鹜文楷、思源黑体、HarmonyOS Sans
- 英文：Inter、SF Pro、JetBrains Mono
- 数字时钟：JetBrains Mono 或 DIN 风格字体

## 8. 数据存储设计

第一阶段建议完全前端化：

- 配置数据写在 `config.js` 或 `data/*.json` 中。
- 用户输入数据保存在 `localStorage`。
- 天气数据来自第三方 API。

### localStorage Key 建议

```text
dashboard.notes
dashboard.clipboardSnippets
dashboard.settings
dashboard.lastRecommendation
dashboard.quoteHistory
```

### 后续可升级方案

如果后面想多设备同步，可以再加入：

- Supabase
- Firebase
- Cloudflare D1
- GitHub Gist
- Notion API

## 9. 技术实现建议

## 9.1 第一版技术栈

如果你正在学习前端，建议第一版不要太复杂：

- HTML
- CSS
- Vanilla JavaScript
- localStorage
- 免费天气 API

这样更容易理解网页基础，也更适合做成可部署的静态站。

## 9.2 第二版可升级技术栈

当基础功能稳定后，可以升级为：

- Vite
- React 或 Vue
- Tailwind CSS
- Zustand / Pinia
- Framer Motion
- Supabase

## 10. 开发路线图

## 10.1 MVP：先做可用版本

目标：打开页面后，能看到核心信息，并能完成基础交互。

- 搭建基础 HTML / CSS 页面结构。
- 完成 Bento Grid 卡片布局。
- 实现当前时间和日期。
- 实现快速书签导航。
- 实现随机箴言。

## 10.2 V1：加入个人气息

目标：让页面从工具变成个人空间。

- 接入天气 API。
- 根据天气切换卡片背景。
- 增加剪贴板快捷语。
- 增加今日份随机推荐。
- 增加便签墙。
- 加入日间 / 夜间主题。

## 10.3 V2：加入趣味和成长感

目标：让页面具有长期使用的反馈。

- 增加像素宠物或动态植物。
- 记录便签或连续访问后触发宠物反馈或植物成长。
- 增加年度进度、阅读进度、倒数日。
- 增加设置面板。
- 支持导入 / 导出本地数据。

## 10.4 V3：多端同步与内容花园

目标：把页面升级成真正的数字花园。

- 引入后端或云数据库。
- 支持多设备同步。
- 增加文章、笔记、书影音记录页面。
- 增加标签系统。
- 增加公开 / 私密内容区分。

## 11. 组件拆分建议

可以把每个卡片都理解成一个独立组件：

```text
WeatherCard
QuoteCard
ClockCard
BookmarkGrid
SnippetCard
PetWidget
RecommendationCard
NoteWall
ThemeToggle
SettingsPanel
```

每个组件尽量只负责一件事：

- 渲染自己的 UI。
- 处理自己的交互。
- 从统一的 storage 工具读取和保存数据。

## 12. 关键交互细节

### 12.1 页面加载时

1. 读取主题设置。
2. 读取本地便签、快捷语和个性化设置。
3. 渲染时间、日期和倒数日。
4. 请求天气数据。
5. 随机或读取今日箴言。
6. 初始化宠物动画。

### 12.2 用户输入时

- 便签编辑后防抖保存。
- 快捷语复制后显示短暂成功提示。
- 推荐结果可以保存为今日推荐。

### 12.3 错误状态

- 天气 API 失败：显示默认天气卡片和提示。
- localStorage 不可用：提示数据无法持久保存。
- 剪贴板复制失败：显示手动复制内容。

## 14. 非目标

第一版不建议做这些功能：

- 复杂登录系统
- 多用户权限
- 复杂 CMS
- 社交分享
- 评论系统
- 大型后端服务

这些会让学习成本迅速升高，也会削弱“个人空间”的轻盈感。

## 15. 第一版验收标准

完成第一版时，可以用下面的问题检查：

- 打开页面后，是否能在 3 秒内看到时间、日期和主要卡片？
- 快速书签是否能正常跳转？
- 箴言是否能随机切换？
- 页面在手机上是否能正常阅读？
- 视觉上是否已经有“这是我的空间”的感觉？

## 16. 后续灵感

- 根据一天中的时间切换问候语：早上、下午、晚上、深夜。
- 每天第一次打开页面时显示一句“今日开场白”。
- 给宠物增加状态：开心、困、专注、摸鱼。
- 便签可以一键归档为灵感、摘录或备忘。`n- 每月自动生成一张“生活小结”卡片。
- 增加隐藏彩蛋，比如连续点击头像触发特殊动画。

## 17. 总结

这个个人主页的核心不是展示“我有多厉害”，而是帮助自己更舒服地进入一天的工作、学习和生活。它应该像一张数字书桌：左边是天气和时间，中间是今天要做的事，右边有一句喜欢的话，角落里还有一只只属于自己的小宠物。

第一版建议从 **时钟 + 书签 + 箴言 + 便签墙** 开始。等这些基础模块稳定后，再逐步加入天气、剪贴板快捷语、随机推荐和像素宠物。这样既能保持学习节奏，也能让项目持续生长。
