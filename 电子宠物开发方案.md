# 云外庭院 · 庭中黑猫开发方案

> **主题定位：** 一只小黑猫，灵感来自《罗小黑战记》中的小黑。它是云外庭院里自由的住客，会在各个卡片之间闲逛、打盹、好奇地拨弄一切。你不需要刻意"养"它——它自己就在那里生活。

---

## 一、为什么是黑猫

项目已有隐性关联：
- 音乐 widget 默认曲目是 **《晚安喵》**（罗小黑战记主题曲）
- 整体风格是"云外庭院"——一只黑猫在云端庭院中漫步，画面天然成立
- 猫是独立的生物，不需要繁琐的喂养操作，符合"陪伴感"而非"负担感"

## 二、功能设计

### 2.1 核心状态

| 状态 | 范围 | 说明 |
|------|------|------|
| 活跃度 (energy) | 0-100 | 决定猫在做什么。高→四处溜达；中→坐下舔毛；低→打盹 |
| 心情 (mood) | 0-100 | 受互动和环境影响。高→翻肚皮、蹭卡片；低→背对屏幕、不理人 |
| 饱腹 (hunger) | 0-100 | 慢速下降，降到极低时猫会消失一阵子（"去觅食了"），回来时恢复 |

**核心思路：状态不强制玩家管理，而是影响猫的行为表现。** 即使完全不互动，猫也会自己活动，只是开心程度不同。

### 2.2 行为模式

猫不固定在某个 widget 里——它是在桌面上**自由行走**的。

```
     ┌───────────────────┐
     │    roaming(闲逛)   │ ←── 默认状态，在卡片之间走动
     └───┬───────┬───────┘
         │       │
         ▼       ▼
   ┌────────┐ ┌────────┐
   │explore │ │  rest  │
   │(探索卡片)│ │(休息)  │
   └───┬────┘ └───┬────┘
       │          │
       ▼          ▼
 ┌──────────┐ ┌──────────┐
 │interact  │ │sleep     │
 │(与卡片互动)│ │(打盹/睡觉)│
 └──────────┘ └──────────┘
```

- **Roaming（闲逛）**：从当前位置走向一个随机目标点，走过时经过卡片附近
- **Explore（探索）**：走到某张卡片前，盯着看、用爪子拨弄
- **Interact（互动）**：对卡片做出具体动作（见下方）
- **Rest（休息）**：原地坐下，舔毛、打哈欠、发呆
- **Sleep（睡觉）**：蜷缩成一团睡觉，Zzz 飘出

### 2.3 与卡片的互动

猫会主动走向各个 widget 进行互动，每种卡片有独特的反应：

| Widget | 猫的行为 |
|--------|---------|
| **👤 档案卡片** (profile) | 跳到头像旁蹲坐，歪头看"主人" |
| **🌍 世界窗口** (portal) | 坐在浮岛场景前，伸爪拍空中飘浮的小点 |
| **👋 问候卡片** (greeting) | 在这里蜷缩睡觉——这是它的"窝" |
| **🕐 时钟卡片** (clock) | 整点时走到时钟前坐着看数字跳动；番茄钟响时竖起耳朵 |
| **📅 日历卡片** (calendar) | 趴在上面，尾巴盖住今天的日期 |
| **🔗 快捷链接** (links) | 在链接之间走来走去，尾巴扫过按钮 |
| **📜 箴言卡片** (quote) | 点击后猫会追着飘出的文字跑 |
| **🎵 音乐卡片** (music) | 播放音乐时——趴在旁边睡；暂停时——抬头看，耳朵动 |
| **🌤 天气卡片** (weather) | 看窗外（朝向 portal 方向），尾巴随天气状态摆动频率变化 |
| **📝 便签** (notes) | 在便签上"踩一脚"留下 🐾 emoji，或趴在便签上 |

### 2.4 互动反馈

用户可以对猫做的操作：

| 操作 | 反馈 |
|------|------|
| **点击猫** | 猫回头看你 → 如果心情好就眯眼"😊"，心情一般就甩尾巴，心情差就扭过头 |
| **长按/拖拽猫** | 猫被拎起来 → 挣扎两下然后"算了你抱吧"的放弃表情 |
| **在猫附近晃动鼠标** | 猫的视线跟随鼠标，像看逗猫棒 |
| **双击猫** | 猫打滚翻肚皮（心情好的时候）或伸懒腰 |

### 2.5 随时间的行为变化

| 条件 | 行为 |
|------|------|
| 白天 (日间模式) | 活跃度高，频繁走动探索 |
| 夜间 (夜间模式) | 更爱睡觉，走动间隔变长；但眼睛变成发光的大瞳孔 |
| 长时间没互动 | 猫会走到屏幕中央坐下看着你，然后背对屏幕 |
| 鼠标静止 > 30s | 猫会走近鼠标指针嗅一嗅，然后趴在旁边 |
| 打开新页面时 | 猫会转头看向新页面方向 |

## 三、视觉设计

### 3.1 猫咪造型（纯 CSS 绘制）

不使用图片资源，用 CSS 画一只小黑猫：

```
      /\    /\
     /  ^  ^  \       ← 两个三角形耳朵 + 圆形脸
    |    ••    |       ← 发光的黄色大眼睛（夜间：放大 + 辉光）
    |    w    |        ← 小嘴，随时间变成 ^ / w / 一 / 口
     \  ———  /
      \_____/
        |  |          ← 前腿（坐下时）
        |  |
       ╱    ╲         ← 尾巴，会左右摆动
      ╱      ╲
```

- **身体**：`border-radius` 组合绘制圆形猫头 + 椭圆身体
- **耳朵**：用 `::before` / `::after` 或单独元素做三角形，内置粉红色内耳
- **眼睛**：黄色圆形（`#FFD700`），夜间扩大并加 `box-shadow` 发光
- **尾巴**：长条形，用 CSS transform 做摆动动画
- **胡须**：用细长伪元素或 `box-shadow` 模拟

### 3.2 尺寸与位置

- 猫本身约 **80×60px**（坐姿），**100×50px**（趴着睡），**70×50px**（走路侧影）
- 猫是一个**独立定位的元素**，不固定在某个 widget 内，使用 `position: fixed` 或 `absolute` 在桌面范围内
- 位置平滑移动（`requestAnimationFrame` + 缓动函数）

### 3.3 状态 → 表情映射

| 状态 | 耳朵 | 眼睛 | 嘴 | 尾巴 |
|------|------|------|-----|------|
| 闲逛 | 竖起 | `••` 正常眨眼 | `w` 放松 | 微翘，慢摆 |
| 好奇（探索卡片） | 向前倾 | `••` 瞪大 | `o` 微张 | 竖起微颤 |
| 开心（被摸/好心情） | 放松 | `◕◕` 半眯 | `^` 微笑 | 竖着，尖勾 |
| 困了 | 耷拉 | `一一` 半闭 | `—` | 垂着无力 |
| 睡着了 | 完全趴下 | `——` 闭合 | `—` | 围住身体 |
| 受惊 | 飞机耳 | `OO` 瞪大 | `O` | 炸毛竖起 |
| 不开心 | 背过去 | `••` 冷淡 | `一` | 快速甩动 |
| 打哈欠 | 往后 | `> <` 紧闭 | `口` 张大 | 不动 |

### 3.4 主题适配

| 元素 | 日间 | 夜间 |
|------|------|------|
| 毛色 | `#2a2a2a`（深灰黑） | `#1a1a2e`（更深的蓝黑） |
| 眼睛 | `#FFD700`，正常大小 | 放大 1.3×，`box-shadow: 0 0 12px #FFD700` 发光 |
| 瞳孔 | 竖线，日常大小 | 极大（适应黑暗） |
| 耳朵内 | `#FFB6C1` 粉 | `#8B5CF6` 荧紫 |
| 尾巴尖 | 黑色 | 微微发光 `#6ee7b7` |

### 4.2 模块职责

#### `cat.js` — 黑猫主控制器

```
CatController
├── state: { energy, mood, hunger, position, target, ... }
├── behaviorFSM: roaming → explore → rest → sleep → ...
├── loadState() / saveState()
├── tick() — 每 2 秒更新行为决策
├── moveTo(x, y) — 平滑寻路
├── setExpression(ears, eyes, mouth, tail) — 切换表情
├── reactToWidget(widgetId) — 触发卡片互动
├── handleClick() / handleDrag() / handleMouseNear()
└── init()
```

**关键数据模型：**

```js
// 持久化存储 (localStorage: 'dashboard.cat')
```

### 4.3 移动系统

```js
// 缓动寻路算法
function moveToward(current, target, speed) {
  const dx = target.x - current.x;
  const dy = target.y - current.y;
  const dist = Math.hypot(dx, dy);
  if (dist < 2) return { x: target.x, y: target.y }; // 到达
  const step = Math.min(speed, dist);
  return {
    x: current.x + (dx / dist) * step,
    y: current.y + (dy / dist) * step,
  };
}

// 行为决策（简化版）
function decideNextAction(state) {
  // 1. 如果猫已经在某个 widget 附近，大概率互动/休息
  // 2. 如果附近有 widget 且心情好，走过去探索
  // 3. 如果精力低，寻找 greeting 卡片睡觉
  // 4. 否则随机漫游到另一个位置
  // 5. 偶尔看向鼠标位置
}
```

猫的移动速度约为 **30-60px/秒**（走路）、**120px/秒**（被逗猫棒吸引追鼠标时）。

### 4.4 Widget 交互检测

扫描所有 widget 的位置并维护一张"兴趣点"列表：

```js
const WIDGET_INTEREST_POINTS = {
  profile: { x: '8%', y: '13%', action: 'sitBeside' },
  greeting: { x: '37%', y: '34%', action: 'sleep' },
  clock: { x: '68%', y: '20%', action: 'watch' },
  calendar: { x: '66%', y: '40%', action: 'lieOn' },
  portal: { x: '36%', y: '7%', action: 'pawAt' },
  links: { x: '34%', y: '67%', action: 'walkThrough' },
  quote: { x: '35%', y: '78%', action: 'chase' },
  music: { x: '57%', y: '78%', action: 'sleepNear' },
  weather: { x: '74%', y: '74%', action: 'sitLook' },
};
```

**判定逻辑：**
1. 猫选择下一个目标时，以一定概率选择某个 widget 附近（权重可调）
2. 到达后，触发 `reactToWidget(widgetId)`，产生对应的猫+卡片动画
3. 在 widget 附近停留 5-15 秒，然后继续 roam

### 4.5 卡片的交互反馈

当猫与卡片互动时，卡片本身也产生微小反馈：

| 卡片 | 反馈效果 |
|------|---------|
| 时钟 | 数字闪烁一下，像被猫拍了一下 |
| 便签 | 文字末尾追加 🐾 |
| 箴言 | 自动切换到一条随机的"猫咪哲学" |
| 音乐 | 暂停时猫靠近→自动续播；播放时猫睡觉→音量减弱 |
| 天气 | 猫看的方向出现小爪印水渍 |
| 快捷链接 | 猫走过时链接按钮轻轻晃动 |
| 日历 | 今天的数字加一个猫爪高亮圈 |

## 六、交互详细设计

### 6.1 点击检测

```js
catElement.addEventListener('click', (e) => {
  e.stopPropagation();
  const mood = state.mood;
  if (mood > 70) {
    // 眯眼开心 → 短暂翻肚皮概率 30%
    setExpression('happy');
    spawnParticles('heart', 5);
    state.mood += 3;
  } else if (mood > 30) {
    // 回头看 → 甩尾巴
    setExpression('lookBack');
    setTimeout(() => setExpression('idle'), 1500);
    state.mood += 1;
  } else {
    // 扭头不理
    setExpression('ignore');
    setTimeout(() => setExpression('idle'), 2000);
  }
  saveState();
});
```

### 6.2 鼠标接近检测

```js
// 猫附近 120px 内有鼠标 → 猫会转头看鼠标方向
catElement.addEventListener('pointermove', (e) => {
  if (!catNearMouse) return;
  const catRect = catElement.getBoundingClientRect();
  const catCenterX = catRect.left + catRect.width / 2;
  const lookDirection = e.clientX > catCenterX ? 'right' : 'left';
  catElement.dataset.look = lookDirection;
  // 耳朵微微转向鼠标方向
});

// 全局鼠标跟踪，距离猫 < 150px 时触发
document.addEventListener('pointermove', (e) => {
  const dist = distance(e.clientX, e.clientY, catX, catY);
  catNearMouse = dist < 150;
  if (dist < 80 && state.energy > 50) {
    // 猫缓慢转向鼠标，像看逗猫棒
  }
});
```

### 6.3 特殊事件

**整点报时：** 监听自定义事件或检测分钟变化，猫走到时钟前坐着看。

**番茄钟结束：** 猫耳朵竖起，头歪一下。

**天气刷新：** 猫走到天气卡片附近，抬头看（朝 portal/天空方向）。

**音乐播放/暂停：**
- 播放 → 猫走到音乐卡片旁边，趴下闭眼
- 暂停 → 猫抬头睁眼，耳朵动一动

## 八、技术要点

### 8.1 性能保障

- **猫移动使用 `requestAnimationFrame` + 缓动函数**，帧率限制在 30fps
- 不移动时 DOM 无变化，零 CPU 开销
- 粒子系统用 CSS animation，不用 JS 循环
- localStorage 写入只在状态跨天变化时进行（不是每秒写）

### 8.2 无障碍

- 猫容器添加 `role="presentation"` 或 `aria-hidden="true"`（装饰性元素）
- 但点击猫的区域用 `aria-label="庭院里的黑猫，点击互动"`

### 8.3 与现有系统集成

| 系统 | 集成点 |
|------|--------|
| `theme.js` | 检测 `data-theme` 变化切换猫的配色 |
| `clock.js` | 监听番茄钟结束事件；整点检测 |
| `music.js` | 监听播放/暂停事件 |
| `weather.js` | 天气变化时猫的反应 |
| `notes.js` | 猫"踩"便签追加 🐾 |
| `quotes.js` | 猫靠近时切换到猫咪相关箴言 |
| `dom.js` | 添加 `cat` 引用 |
| `config.js` | 添加 `dashboard.cat` 存储 key |

### 8.4 多页面策略

- 首页：完整版黑猫，全尺寸走动
- 子页面（projects/articles/photos/favorites）：**猫缩小到 50%** 放置在右下角，仅保留 idle 动画和点击反应
- 子页面猫的位置存储在 `sessionStorage`，回到首页时恢复

### 8.5 数据格式

```js
  behaviorState: 'roam',
  stats: {
    daysLived: 1,
    totalDistance: 0,      // 累计行走像素
```

## 十、方案对比（原庭灵 vs 黑猫）

| 维度 | 庭灵方案（v1） | 黑猫方案（v2）✅ |
|------|---------------|----------------|
| 形象风格 | 抽象光球精灵 | 具象黑猫，有角色认知基础 |
| 项目契合度 | 还可以 | **极高**——已有罗小黑音乐关联 |
| 互动方式 | 面板按钮操作 | 自然观察+点击+猫咪自主行为 |
| 开发成本 | 需要粒子系统、游戏等较多 | CSS 绘制猫相对简单，行为逻辑为主 |
| 用户情感连接 | 一般 | **强**——用户对猫有天然情感投射 |
| 卡片互动 | 作为独立 widget 存在 | **猫在卡片之间走动**，互动更自然 |
